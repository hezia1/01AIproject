# 六模块可选接入设计

PPT 第三页的六个能力模块在平台中设计为可插拔模块。用户创建或配置项目时，可以选择接入哪些模块。每个模块独立启用、独立配置、独立执行任务，但结果统一进入安全知识中枢和 ASPM 治理闭环。

## 1. 模块清单

### SAST · 智能静态审计

对应 PPT 能力：

- 定制化安全 Skill
- 多 Sub-agent 编排
- 行业历史漏洞知识库

平台职责：

- 代码仓库扫描。
- 规则扫描与 AI 审计。
- 多 Agent 分工复核。
- 输出代码漏洞、证据、解释和修复建议。

### SCA · 供应链风险分析

对应 PPT 能力：

- SBOM 生成
- 组件漏洞匹配
- 许可证风险归一化
- 直接/传递依赖分析
- 版本归一化与影响分析
- 多工程生态依赖解析

平台职责：

- 解析依赖文件。
- 生成组件清单。
- 匹配漏洞、许可证和恶意组件风险。
- 输出修复优先级。

### AGENT · Agent 供应链安全

对应 PPT 能力：

- 指令文件扫描
- 工具协议扫描
- 插件扩展扫描
- 规则检测、AI 审计、覆盖矩阵与信任评分
- 私有源接入、配置解析、公开只读报告与复测能力

平台职责：

- 扫描 Agent 指令、MCP、工具定义和插件清单。
- 识别提示注入、工具滥用、敏感资源访问和密钥读取风险。
- 生成信任评分和复测报告。

### DAST · 漏洞动态验证

对应 PPT 能力：

- Web 业务验证
- 静态发现联动验证
- 策略生成
- 三色裁决：可利用 / 不确定 / 不可利用
- 执行日志、证据归档、验证过程追踪与报告生成

平台职责：

- 将 SAST/SCA/Agent 发现转换为验证策略。
- 对目标 Web 应用执行验证。
- 输出三态裁决和证据链。

### SANDBOX · 沙箱动态证据链

对应 PPT 能力：

- 隔离环境
- 行为监控
- 运行时覆盖
- 调用账本
- 多类 Agent 运行时适配
- 策略化探测
- 结构化工具调用采集
- AI 驱动动态验证能力

平台职责：

- 在隔离环境运行目标程序、插件或 Agent。
- 采集文件、网络、进程、环境变量和工具调用行为。
- 为 DAST 和 Agent 安全提供动态证据。

### ASPM · 平台治理与交付

对应 PPT 能力：

- 项目组、跨项目关联、攻击链、风险趋势与整改闭环
- 开放工具接口、批量任务、研发流水线安全门禁
- 模块权限、授权配额、审计日志、诊断导出与合规报告

平台职责：

- 聚合各模块结果。
- 管理整改、复测、关闭和误报。
- 输出治理看板、合规报告和安全门禁结果。

## 2. 模块关系

- SAST、SCA 是优先落地的基础检测模块。
- ASPM 是治理入口，建议默认启用。
- DAST 依赖静态或供应链发现，优先联动 SAST。
- SANDBOX 优先服务 Agent 模块，提供运行时证据链。
- 安全知识中枢不是用户可选模块，而是六模块共享底座。
